将所有的动作，反馈，结果类型进行定义
主要内容包括：
    交互：用户：需求，prompt；api：目标与进度，原因解释，文档生成
    文件操作的请求与输入
        单文件操作（读取展示内容在最大长度限制下，读取头和尾各占一半的量）
            添加
            删除
            修改
            读取（支持py，txt,csv,md，json，html，css等常见可打开文件，并且获取每个文件的大小信息，
                并且对文件读取的字符数进行控制（如文件大小不超过20k，超过的只读取前n行），避免数据类文本占用空间造成冗余）
        目录操作
            获取目录下树结构（展示内容限制总长度，不超过300，避免重复冗余，如数据库文件）
            创建目录
            删除目录

    程序执行与管理的请求与输入
        程序执行
        程序监控

    to_json
        所有类对象均可将属性转为标准json结构
    parse_json
        将json结构文本解析，转为对应的实例化对象
